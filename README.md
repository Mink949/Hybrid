# 🌡️ Regime-Shift Recovery Hybrid: Weather-Mortality Modeling

[![Version](https://img.shields.io/badge/version-v13-blue.svg)](hybrid.py)
[![Python](https://img.shields.io/badge/python-3.8+-green.svg)](https://www.python.org/)

A sophisticated hybrid modeling system designed to capture the non-linear relationship between weather patterns and mortality rates in Australia (NSW, VIC, QLD), while accounting for the temporary regime shift caused by the COVID-19 pandemic.

## 🌟 Overview

This project implements a multi-stage modeling approach to separate long-term mortality trends from acute weather-driven fluctuations. The "v13" upgrade introduces a **Decaying COVID Regime Index**, which models the pandemic's impact as a temporary shock that naturally fades over time, rather than a permanent structural break.

## 🏗️ Core Architecture

The system employs a **3-Stage Hybrid Approach**:

1.  **Stage 1: Baseline Prediction**
    - Uses **Ridge Regression** with **Spline Transformers** for time-based trends and seasonal components (`week_sin/cos`).
    - Incorporates the `covid_decay` feature to model the post-2022 mortality regime.
    - Captures autoregressive signals from lagged death counts.

2.  **Stage 2: Weather Residual Modeling**
    - A **LinearGAM** (Generalized Additive Model) targets the residuals from Stage 1.
    - Uses **Tensor Spline Interactions** between temperature anomalies and humidity.
    - Captures non-linear effects of rainfall, solar radiation, and temperature variability.

3.  **Stage 3: Hybrid Integration & Inference**
    - Combines predictions from both stages to produce the final mortality estimate.
    - Implemented with state-specific optimizations for NSW, VIC, and QLD.

## 📊 Key Features

-   **`covid_decay`**: An exponential decay function ($\exp(-0.03 \cdot \text{weeks})$) modeling the recovery from the COVID-19 shock.
-   **`temp_anomaly`**: Weekly temperature deviations from a 52-week moving average.
-   **Cross-Basis Splines**: Capturing delayed (lagged) effects of temperature on mortality.
-   **Chameleon Architecture**: Automatically adjusts spline complexity based on state-specific data density.
